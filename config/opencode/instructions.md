When answering technical questions, use web search or web fetch early if:

- the answer may depend on current versions, APIs, package names, documentation, releases, compatibility, or known bugs
- the user asks about an external tool, package, library, service, wiki, or error message
- local reasoning would involve guessing about current behavior
- an authoritative lookup would be faster than inferring

For local code and config tasks, inspect local files first. For current docs, package behavior, unfamiliar tools, and external errors, proactively look things up instead of spending a long time reasoning from memory.
